Replit AI 요청용 프롬프트
Kastor Detective Academy 프로젝트에서, 기존 Evidence Notebook(Characters/Data/Talks/Photos 탭 형태)을 “노드형 증거 보드” UI로 전환해 주세요. 데스크톱과 모바일 모두에서 작동해야 합니다. 다음 요구사항을 모두 반영하세요.## 1. 기본 구조- Evidence Notebook을 열면 탭 대신 “Evidence Board” 뷰가 기본으로 표시되도록 합니다.- 보드는 전체 화면을 덮는 캔버스(흰색 또는 종이 질감 배경) + 노드/연결선으로 구성합니다.- 기존 데이터(캐릭터, 자료, 대화, 사진 등)는 노드 정보로 변환하고, JSON 형태로 저장하거나 기존 스토어를 재활용하세요.## 2. 노드 구성- 각 증거 항목을 하나의 노드로 표현합니다.  - 캐릭터: 원형 노드, 캐릭터 얼굴 그림(이미지 또는 이모지) + 이름.  - 데이터(차트, 테이블): 사각형 노드, 아이콘(📊 등) + 제목.  - 대화/로그: 말풍선 모양 또는 사각형 노드, “Kastor & Chris 대화 로그” 등 제목.  - 사진: 폴라로이드 스타일 노드(미니 썸네일 + 제목).- 노드를 탭/클릭하면 하단에 Bottom Sheet(모바일) 또는 오른쪽 패널(데스크톱)로 상세 내용을 보여줍니다.  - 상세 정보에는 기존 탭에서 보여주던 텍스트/이미지/차트 링크를 그대로 활용하세요.## 3. 연결선(Edges)- 노드 간 관계를 화살표 또는 실선으로 연결합니다.  - 예: 데이터 → 캐릭터 (이 데이터가 해당 캐릭터를 의심하게 만듦).  - 연결선 위에 짧은 텍스트(“야근 14시간” 등) 또는 아이콘을 추가.- 사용자가 특정 노드를 선택하면 그 노드와 직접 연결된 선/노드만 진하게, 나머지는 반투명 처리하여 가독성을 높입니다.## 4. 제스처 & 인터랙션- 데스크톱:  - 마우스 드래그로 보드 패닝.  - Ctrl/Command + 마우스 휠 혹은 UI 버튼(+/-)로 줌 인/아웃.  - 노드 드래그로 위치 조정 가능(자유 배치).- 모바일:  - 핀치 줌으로 확대/축소.  - 손가락 드래그로 패닝.  - 노드를 탭하면 Bottom Sheet가 올라오고, 시트를 닫으면 보드가 다시 전체 표시.- 노드를 새로 발견할 때(스토리 진행 중), 노드가 깜박이거나 애니메이션으로 등장.## 5. 보조 UI- 보드 상단 또는 우측 상단에 간단한 컨트롤러 제공:  - “Reset View”(전체 보기), “List View”(기존 탭 형태 리스트로 전환).  - 필터 버튼(캐릭터/데이터/대화/사진 별로 온/오프).- 하단 또는 상단에 현재 목표 텍스트(예: “용의자와 연결된 증거 찾기”) 표시.- 기존 Score/Evidence/Hints 카운터는 보드 상단의 헤더에 유지.## 6. 데이터 흐름- 기존 증거 데이터를 가져오는 함수 또는 스토어를 재사용하여 Evidence Board 컴포넌트에서 노드/엣지 구조로 변환합니다.- 변환 예시:  
{
id: "node_chris",
type: "character",
label: "Chris",
avatar: "/assets/chris.png",
connections: [
{ target: "node_overtime_chart", label: "14시간 야근" }
]
}
- 스토리 진행(씬 완료 등)에 따라 노드가 추가되거나 업데이트되도록 hooks 또는 상태 관리를 설정하세요.## 7. 반응형 디자인- 데스크톱(≥1024px): 전체 보드 + 오른쪽 상세 패널 레이아웃.- 태블릿/모바일(<1024px): 보드 전체 화면 + 노드 탭 시 bottom sheet로 상세 정보 표시.- 최소 폰트 크기, 노드 크기를 고려하여 모바일에서도 읽기 편하도록 조정.## 8. 사용성 & 성능- 노드가 많아질 경우를 대비해 가벼운 레이아웃 알고리즘(예: force layout 또는 고정 레이아웃)을 적용하되, 성능이 떨어지지 않게 최적화합니다.- 줌 단계를 제한하거나, 노드를 단계적으로 표시(처음엔 핵심 6~8개, 진행에 따라 추가)하여 복잡도를 조절.## 9. 테스트 & 문서화- 노드 클릭 → 상세 정보 표시, 필터 기능, 줌/패닝이 데스크톱과 모바일에서 제대로 작동하는지 확인.- 새 증거가 추가되었을 때 보드에 반영되는지 테스트.- README 또는 코드 주석에 Evidence Board 사용 방법을 간단히 추가.위 요구사항이 모두 충족되도록 Evidence Notebook UI를 개편해 주세요. 필요하다면 기존 컴포넌트 구조를 수정하거나 새로 작성해도 좋습니다.
위 프롬프트를 Replit AI에게 전달하면, 노드형 증거판을 구현하면서도 기존 데이터 구조와 스토리 시스템을 유지하도록 유도할 수 있습니다.
